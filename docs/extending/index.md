# Extending the Benchmark

The Variational Learning Benchmark is designed to be highly modular and extensible. Depending on which part of the benchmark you are running, there are slightly different patterns for adding new components.

Please refer to the following guides:

- **[In-Domain and Standard Tasks](indomain.md)**: How to add new Datasets, Methods (Optimizers), and Models to the standard training and evaluation pipelines.
- **[Federated Learning Tasks](federated.md)**: How to add new partitioned datasets, federated aggregation methods, and specific federated models to the decentralized training pipeline.
- **[Text Generation](text_generation.md)**: How to add custom models, uncertainty optimizers, and scoring metrics to the text generation evaluation pipeline.
